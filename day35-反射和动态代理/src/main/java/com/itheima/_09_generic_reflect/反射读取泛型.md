# 反射读取泛型：getGenericType() 说明

## 一、它是什么

`getGenericType()` 是 `java.lang.reflect.Field` 的方法，作用是：**拿到字段声明时"带泛型"的完整类型信息**，返回一个 `java.lang.reflect.Type`。

同一系列的方法还有（作用相同，只是位置不同）：

| 方法 | 所属 | 读取的位置 |
| --- | --- | --- |
| `getGenericType()` | `Field` | 字段声明的泛型 |
| `getGenericReturnType()` | `Method` | 方法返回值的泛型 |
| `getGenericParameterTypes()` | `Method` | 方法参数的泛型 |
| `getGenericSuperclass()` | `Class` | 父类继承处的泛型 |

---

## 二、和 getType() 的区别（核心）

以字段 `List<User> users` 为例：

| 方法 | 返回 | 结果 |
| --- | --- | --- |
| `f.getType()` | `Class<?>` | 只得到 `List`（泛型被擦除） |
| `f.getGenericType()` | `Type` | 得到 `java.util.List<...User>`（保留了 `<User>`） |

一句话：`getType()` 拿"擦除后的原始类型"，`getGenericType()` 拿"声明时写的泛型类型"。

---

## 三、为什么类型擦除了还能拿到

Java 泛型有**类型擦除**：`new ArrayList<User>()` 和 `new ArrayList<String>()` 在运行期是同一个类，实例里没有 `<User>` 信息。

但**擦除的是"实例数据"，不是"声明"**。写在下面几个位置的泛型实参，会被编译器保存进字节码的 **Signature 属性**：

- 字段声明：`private List<User> users;`
- 方法签名：返回值、参数
- 父类 / 接口继承：`class A extends B<User>`

`getGenericType()` 等方法读的就是 Signature 属性里的这份信息，所以能还原出 `List<User>`。

---

## 四、返回的 Type 怎么用

`Type` 是顶层接口，常见的实现类型：

| Type 的实际类型 | 对应写法 | 举例 |
| --- | --- | --- |
| `Class` | 普通类型，无泛型 | `String name` |
| `ParameterizedType` | 参数化类型，带 `<>` | `List<User>` |
| `TypeVariable` | 类型变量 | `T` |
| `WildcardType` | 通配符 | `? extends Number` |
| `GenericArrayType` | 泛型数组 | `T[]`、`List<User>[]` |

判断并取出尖括号里的实参：

```java
Type type = f.getGenericType();

if (type instanceof ParameterizedType) {          // 说明是 List<User> 这种带 <> 的
    ParameterizedType pt = (ParameterizedType) type;
    Type raw   = pt.getRawType();                 // List
    Type[] args = pt.getActualTypeArguments();    // [User]  ← 尖括号里的实参
}
```

> 注意：字段若只是 `String name`（无泛型），`getGenericType()` 返回的是普通 `Class`，**不是** `ParameterizedType`，因此要先用 `instanceof ParameterizedType` 判断，否则无实参可取。

---

## 五、示例执行流（对照 Demo）

```java
Class<UserRepository> clazz = UserRepository.class;

// 1. 读字段泛型
for (Field f : clazz.getDeclaredFields()) {
    Type type = f.getGenericType();               // 若字段是 List<User> → 得到 List<User>
    if (type instanceof ParameterizedType) {
        ParameterizedType pt = (ParameterizedType) type;
        for (Type arg : pt.getActualTypeArguments()) {
            System.out.println("实参: " + arg.getTypeName());   // User
        }
    }
}

// 2. 读方法返回值泛型（原理与字段完全一样）
Method m = clazz.getDeclaredMethod("findUsers", int.class, int.class);
Type retType = m.getGenericReturnType();
```

---

## 六、有什么用（实际价值）

`getGenericType()` 是很多框架能"看懂泛型"的底层依据：

- **Gson / Jackson**：反序列化时知道要构造 `List<User>` 而不是单个 `User`（`TypeToken`、`TypeReference` 就是靠它）。
- **Spring**：`@Autowired List<Handler>` 能注入所有实现，靠的就是读取泛型实参。
- **MyBatis**：把结果集映射成 `List<User>`。

---

## 七、小结

- `getGenericType()`＝读**字段声明里写的泛型**；`getType()`＝读**擦除后的原始类型**。
- 类型擦除擦掉的是运行期对象里的泛型，但**声明位置的泛型保留在字节码 Signature 属性**里，反射能读回来。
- 返回的 `Type` 若是 `ParameterizedType`，用 `getActualTypeArguments()` 取出 `<>` 里的实参。
- 这是 Gson、Jackson、Spring 等框架识别 `List<User>` 的关键机制。
