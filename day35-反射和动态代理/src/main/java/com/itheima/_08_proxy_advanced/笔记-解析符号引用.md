# 笔记：解析符号引用（Resolution）

> 补充自"类加载三步"中链接（Linking）阶段的第三小步。

一句话：**把 `.class` 里"用名字/文字描述的引用"，替换成"能直接定位到内存的引用"**，并顺带做存在性和权限校验。

---

## 一、两个核心概念

### 符号引用（Symbolic Reference）
`.class` 字节码里对其它类、方法、字段的引用**不是内存地址，而是一串文字描述**。
因为编译期根本不知道运行时它们会被加载到内存哪个位置，只能先用"名字"记着（存在常量池里）。

例如源码 `System.out.println(...)` 编译后，常量池里存的是：
```
类:   java/lang/System
字段: out，类型 Ljava/io/PrintStream;
方法: java/io/PrintStream.println:(Ljava/lang/String;)V
```
这些"用字符串描述我要找谁"的东西，就是符号引用——只是个名字，还没跟真实内存挂钩。

### 直接引用（Direct Reference）
能**直接指向内存中目标**的东西：方法的实际入口地址、字段在对象里的偏移量、目标 `Class` 的指针等。

---

## 二、解析做的操作

把"名字"翻译成"内存里的实际位置"：

| 符号引用（翻译前） | 直接引用（翻译后） |
|---|---|
| 类名字符串 `"java/lang/System"` | 已加载的 `System` 类在方法区的地址 |
| 字段 `System.out` | 该字段在内存里的具体位置/偏移 |
| 方法 `PrintStream.println` | 该方法的实际入口地址 |

分几类：**类/接口解析、字段解析、方法解析**。
过程中还会**做检查**：目标类是否存在、字段/方法是否真的有、访问权限够不够
（如引用了 `private` 却访问不到，会报 `IllegalAccessError`）。

---

## 三、直观类比

符号引用像**通讯录里写的"张三"这个名字**；
解析就是**把"张三"换成他真实的手机号**。
翻译之前只知道要找张三，翻译之后才能真正拨通。

---

## 四、为什么需要这一步

Java 是**分别编译、运行时才把各个类拼到一起**的。编译某个类时它引用的
`System`、`String` 还没加载，不可能知道运行时它们在内存哪。
所以先记名字（符号引用），等相关类都加载进来后，再统一把名字换成真实地址（直接引用）。
这就是 Java **动态链接**的体现。

---

## 五、易混点：何时解析

解析**不一定在初始化前一次性做完**。JVM 允许**惰性解析（延迟解析）**——
很多符号引用是等到**第一次真正用到那条指令时**才去解析
（如第一次调用某方法、第一次访问某字段）。
放在链接阶段是标准说法，但实际运行中常常是"用到才翻译"，并非死板地在初始化前全部翻完。

---

## 一句话总结

**解析符号引用 = 把字节码里"按名字找人"的引用，换成"能直接定位到内存"的引用，并顺带做存在性和权限校验。**
