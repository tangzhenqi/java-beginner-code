# 泛型方法（Generic Method）声明语法

## 一、以 `unmodifiableMap` 为例拆解

```java
public static <K, V> Map<K, V> unmodifiableMap(Map<? extends K, ? extends V> m)
```

```
public static  <K, V>  Map<K, V>  unmodifiableMap
   ↑       ↑     ↑         ↑           ↑
 访问修饰  静态  类型参数   返回值类型    方法名
                声明
```

## 二、关键点：`<K, V>` 出现在返回值前面

最容易困惑的就是这个 `<K, V>`，它夹在 **`static` 和返回值类型之间**。
这个位置的 `<K, V>` 是在**声明（定义）**类型参数，意思是：

> 「我这个方法里要用到两个还没确定的类型，我先给它们起名叫 `K` 和 `V`。」

声明之后，后面才能使用它们：

```java
public static <K, V> Map<K, V> unmodifiableMap(Map<? extends K, ? extends V> m)
//            ╰─声明─╯ ╰─用作返回值─╯              ╰────── 用作参数类型 ──────╯
```

## 三、为什么必须先声明 `<K, V>`？（与类级泛型对比）

```java
// ① 类级泛型：K、V 在“类名后面”声明，整个类共用
class HashMap<K, V> {
    public V get(K key) { ... }   // 方法里直接用 K、V，不用再声明
}

// ② 方法级泛型：K、V 只属于这一个方法，所以要在方法自己身上声明
public static <K, V> Map<K, V> unmodifiableMap(...) { ... }
//            ↑ 这里声明，因为它是 static 方法，不属于任何对象，
//              无法用类的泛型参数，只能自己声明一套
```

要点：**静态方法不能使用类的泛型参数**。`Collections`、`ListUtil` 这类工具类方法全是
`static`，所以每个方法只能自己声明需要的类型参数。

## 四、调用时类型怎么确定？——自动推断

调用时**不需要手动传** `K`、`V`，编译器根据传入参数**自动推断（type inference）**：

```java
Map<String, Integer> raw = new HashMap<>();

Map<String, Integer> m = Collections.unmodifiableMap(raw);
//                                                     ↑
//   编译器看到 raw 是 Map<String,Integer>，
//   于是推断出 K = String，V = Integer
```

如需显式指定（极少用），把类型放在方法名前的点后面：

```java
Map<String, Integer> m = Collections.<String, Integer>unmodifiableMap(raw);
//                                   ╰── 显式指定 K、V ──╯
```

## 五、一句话总结

| 写法 | 含义 |
|---|---|
| `<K, V>`（在返回值前） | **声明**这个方法自己的类型参数 |
| `Map<K, V>`（返回值） | **使用**刚声明的 `K`、`V` |
| 调用时 | 编译器自动推断，不用手填 |

## 六、对照本包的 `ListUtil` / `Demo03_GenericMethod`

本包 `ListUtil` 里的泛型方法就是同一套语法，例如：

```java
// 单类型参数 T
public static <T> void addAll(ArrayList<T> list, T... elements) { ... }
//            ↑声明  ↑返回值void        ↑使用T

public static <T> void swap(ArrayList<T> list, int i, int j) { ... }
```

调用见 `Demo03_GenericMethod.main`：

```java
ListUtil.addAll(strs, "aaa", "bbb");  // 推断 T = String
ListUtil.addAll(nums, 1, 2, 3);        // 推断 T = Integer
```

> 单类型参数写 `<T>`，多类型参数写 `<K, V>`，原理完全相同：**先在返回值前声明，再在方法签名里使用**。
